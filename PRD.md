# 产品需求文档 (PRD) — 今日热搜

> 版本：v1.0 | 日期：2026-05-28 | 状态：MVP 阶段

## 1. 产品概述

### 1.1 产品定位

一个多平台热搜聚合网站，以卡片网格布局展示微博、知乎、百度等平台的热搜榜单，用户可通过链接访问查看实时热点话题。

### 1.2 目标用户

- 想快速了解各平台热点的普通网民
- 关注舆情和话题趋势的内容创作者
- 开发者、学习者（参考架构）

### 1.3 核心价值

在一个页面同时看到多个平台的热搜，省去逐一打开各 App/网站的麻烦。

## 2. 功能需求

### 2.1 P0 — MVP 必备（第一期）

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|----------|
| F1 | 多平台卡片展示 | 首页以卡片网格展示 ≥3 个平台的热搜 Top 10 | 微博、知乎、百度三平台卡片同时可见 |
| F2 | 榜单条目展示 | 每条显示排名、标题、链接 | 点击标题可跳转到源平台 |
| F3 | 热度显示 | 部分平台显示热度数值 | 知乎显示 hot 值，百度显示热度，微博可选 |
| F4 | 真实数据 | 数据来源于平台公开接口，非 Mock | 刷新后数据与源平台一致 |
| F5 | 后端缓存 | 后端 5-10 分钟缓存，减少对源站请求 | 同一分钟内两次请求返回相同数据 |
| F6 | HTTPS 部署 | 部署到公网，支持 HTTPS 访问 | 浏览器无安全警告 |

### 2.2 P1 — 体验增强（第二期）

| 编号 | 功能 | 描述 |
|------|------|------|
| F7 | 自动刷新 | 前端每 5 分钟自动拉取最新数据 |
| F8 | 排名变化 | 显示较上一轮的排名涨跌（↑↓箭头） |
| F9 | 历史趋势 | 点击话题查看 24 小时热度曲线 |
| F10 | 响应式布局 | 移动端适配，卡片自动换行 |
| F11 | 平台切换 | 顶部 Tab 切换单独查看某个平台的完整榜单 |

### 2.3 P2 — 远期规划

| 编号 | 功能 | 描述 |
|------|------|------|
| F12 | 平台扩展 | 接入 B 站、抖音、头条等更多平台 |
| F13 | 热搜推送 | 邮件/微信/Telegram 订阅通知 |
| F14 | 跨平台对比 | 同一话题在不同平台的热度对比 |
| F15 | 暗色模式 | 支持亮/暗主题切换 |

## 3. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 首页加载 < 2 秒，API 响应 < 500ms |
| 可用性 | 单个平台接口失效不影响其他平台展示 |
| 可维护性 | 新增平台只需添加一个适配器文件 |
| 安全 | 仅展示公开数据，不存储用户信息，HTTPS 传输 |
| 合规 | 页脚注明数据来源，声明非商业用途 |

## 4. 范围边界

### 4.1 本期做

- 微博 + 知乎 + 百度 三个平台
- 卡片网格首页
- 后端代理 + 缓存
- 公网部署

### 4.2 本期不做

- 用户注册/登录
- 评论、收藏、分享
- 搜索功能
- 内容推荐/个性化
- 移动端原生 App（仅响应式 Web）
- 全量历史数据回溯（从上线当天开始积累即可）

## 5. 数据合规说明

- 数据来源为各平台**公开**热搜榜单，不抓取用户隐私数据
- 仅展示标题、热度、源链接，不缓存或二次分发正文内容
- 页脚标注"数据来源于各平台公开热搜，仅供学习参考"
- 采集频率控制在 5-10 分钟，避免对源站造成压力
- 遵循各平台 robots.txt 约定
