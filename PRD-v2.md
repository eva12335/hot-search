# 产品需求文档 (PRD) — AI 热搜 V2.0

> 版本：v2.0 | 日期：2026-05-30 | 状态：规划中 | 方向确认：AI 垂直热搜

## 1. V1.0 回顾

### 1.1 已完成

| 功能 | 状态 |
|------|------|
| 微博/知乎/百度三平台卡片展示 | 已上线 |
| 排名、标题、热度、源链接 | 已上线 |
| 后端 5 分钟缓存 + 定时刷新 | 已上线 |
| Render + Vercel 公网部署 | 已上线 |

### 1.2 V1.0 已知局限

| 问题 | 影响 |
|------|------|
| 全品类无筛选，AI 相关内容淹没在娱乐/时政中 | 想看 AI 趋势的用户找不到价值 |
| 无历史数据 | 看不到热度变化趋势，刷新后旧数据丢失 |

| Render 15 分钟休眠 | 长时间无人访问后第一次打开慢 |
| 无排名变化 | 看不到"刚升了还是降了" |
| 无 AI 原生数据源 | 缺少 HuggingFace、GitHub 等专业 AI 热度数据 |

## 2. V2.0 定位变更

**从"全品类多平台热搜聚合"转型为"AI 垂直热搜聚合"。**

- 保留微博/知乎/B 站，按 AI 关键词过滤，追踪 AI 话题在社交平台的讨论热度
- 新增 HuggingFace、GitHub Trending 两个 AI 原生数据源
- 百度适配器移除，替换为 AI 原生数据源
- 不覆盖其他垂直领域（ISO、瑜伽等经调研无可靠数据源，不做）

## 5. 架构变更

### 5.1 新增数据库

V1.0 仅用 NodeCache 内存缓存，V2.0 引入轻量数据库存储历史快照：

```
┌─────────────┐    ┌──────────────┐    ┌───────────┐
│ 适配器采集   │ → │ NodeCache    │ → │ API 响应   │
│ (5个适配器)  │    │ (5min TTL)   │    │ (不变)     │
└─────────────┘    └──────┬───────┘    └───────────┘
                          │ 每次采集后写入
                   ┌──────▼───────┐
                   │ SQLite       │
                   │ (历史快照)    │
                   │ 按天分表      │
                   └──────────────┘
```

- **SQLite** 替代 PostgreSQL（MVP 阶段零运维，单文件存储）
- 每平台每天一张快照表，自动清理 7 天前数据
- 历史趋势 API：`GET /api/hot/:platform/history?hours=24`

### 5.2 适配器目录

```
backend/src/adapters/
├── weibo.ts          (已有，增加 AI 关键词过滤)
├── zhihu.ts          (已有，增加 AI 关键词过滤)
├── baidu.ts          (删除，不再注册)
├── bilibili.ts       (新增，AI 关键词过滤)
├── huggingface.ts    (新增，AI 原生)
├── github-trending.ts (新增，AI 原生)
└── filter.ts         (新增，AI 关键词过滤工具模块)
```

每个适配器遵循相同的 `PlatformAdapter` 接口，零耦合。

### 5.3 API 变更

| 端点 | 变更 |
|------|------|
| `GET /api/hot/all` | 返回 5 个平台（weibo/zhihu/bilibili/huggingface/github-trending） |
| `GET /api/hot/:platform` | 不变 |
| `GET /api/health` | 不变（心跳保活用） |
| `GET /api/hot/:platform/history?hours=24` | **新增** |

### 7.4 布局（响应式）

| 断点 | 列数 | 间距 | 容器内边距 |
|------|------|------|-----------|
| 桌面（> 1024px） | 3 列 | 28px | 24px |
| 平板（641–1024px）| 2 列 | 24px | 24px |
| 手机（≤ 640px） | 1 列 | 20px | 16px |

最大内容宽度：`1280px`，居中。5 张卡片在 3 列布局下自动换行。

### 7.5 卡片效果

- **玻璃拟态**：`backdrop-filter: blur(24px)` + 半透明背景 + 细边框
- **3D 鼠标视差**：`perspective: 1200px`，卡片跟随鼠标位置计算 `rotateX/rotateY`（范围 ±4°/±5°），离开窗口时复位
- **Hover 浮升**：`translateY(-6px)` + 阴影加深 + 顶部边缘光
- **卡片内列表项 hover**：金色微背景 `rgba(201,169,110,0.06)` + 右移 4px

### 7.6 排名徽章

| 排名 | 样式 | 颜色 |
|------|------|------|
| #1 | 渐变底色 + 发光 shadow | 香槟金 `#c9a96e → #e0c88c`，发光 `rgba(201,169,110,0.4)` |
| #2 | 渐变底色 | 银 `#a0a0a0 → #c0c0c0` |
| #3 | 渐变底色 | 铜 `#b87351 → #cd8b6a` |
| #4+ | 暗色底 | `rgba(255,255,255,0.04)` |

前 3 名文字颜色 `#060b1a`（反色），标题加粗。

### 7.7 排名变化指示器

| 变化 | 符号 | 颜色 |
|------|------|------|
| 上升 | `↑` | `#4ade80` |
| 下降 | `↓` | `#f87171` |
| 不变 | `→` | `#5c5a55` |
| 新上榜 | `●` | `#fbbf24`（黄色） |

### 7.8 趋势图面板

- **触发**：点击话题条目
- **呈现**：全屏遮罩（`backdrop-filter: blur(8px)`）+ 居中弹窗
- **弹窗**：最大宽度 `680px`，圆角 `20px`，背景 `#0a1228`
- **图表**：Canvas 2D 绘制，`260px` 高（移动端 `200px`）
  - 面积渐变填充（品牌色 25% → 2% opacity）
  - 折线 + 发光底层双重描边
  - 当前数据点白色外圈 + 品牌色内圈
  - Y 轴热度标注，X 轴 24 小时刻度
  - 支持 HiDPI（devicePixelRatio）
- **元数据**：显示平台、当前排名、热度峰值
- **关闭**：点击遮罩 / ✕ 按钮 / 按 Escape

### 7.9 环境背景

- **星场**：CSS `radial-gradient` 多层点阵，固定定位，金色/白色
- **星云**：3 个 `600px` 模糊圆形（`blur: 120px`，`opacity: 0.12`），分别为橙、粉、靛蓝
- **扫描线**：`repeating-linear-gradient` 半透明条纹，间距 2px

### 7.10 平台 Tab

- **样式**：胶囊按钮组（`border-radius: 999px`），玻璃背景
- **激活态**：金色半透明底色 + 金色文字 + 发光
- **选项**：全部 / 社交平台（微博/知乎/B 站）/ AI 原生（HuggingFace/GitHub）
- **行为**：切换时过滤卡片网格显示

### 7.11 骨架屏

- 暗色占位块（`rgba(255,255,255,0.04)`），`shimmer` 动画闪烁（opacity 0.3→0.6）

### 7.12 技术要求

- 纯 CSS 实现星场、星云、扫描线背景（不依赖图片/Canvas）
- 3D 视差用 CSS Transform（`rotateX/rotateY`），不引入 Three.js
- 趋势图用 Canvas 2D API，不引入图表库
- 所有效果在移动端可降级（减少模糊层级）

## 8. V1.0 vs V2.0 差异分析

### 8.1 产品定位

| 维度 | V1.0 | V2.0 |
|------|------|------|
| 定位 | 全品类多平台热搜聚合 | **AI 垂直热搜** |
| 目标用户 | 泛资讯浏览者 | AI 从业者、开发者、科技爱好者 |
| 内容相关性 | 无保证，娱乐/时政为主 | AI 关键词过滤 + 原生数据源，≥ 80% 相关性 |
| 核心价值 | "看看现在什么热" | "AI 领域现在什么热、什么在升、趋势怎样" |

### 8.2 平台矩阵

| 维度 | V1.0 | V2.0 |
|------|------|------|
| 平台数量 | 3 | **5** |
| 平台列表 | 微博、知乎、百度 | 微博、知乎、B 站、**HuggingFace**、**GitHub Trending** |
| 内容来源 | 全部综合平台 | 3 社交平台（AI 过滤）+ 2 AI 原生平台 |
| 百度 | 保留（偏时政） | **移除**（与 AI 定位不符） |
| B 站 | 无 | **新增**（视频类 AI 内容讨论） |
| HuggingFace | 无 | **新增**（AI 模型/数据集热度） |
| GitHub Trending | 无 | **新增**（AI 开源项目热度） |

### 8.3 数据层

| 维度 | V1.0 | V2.0 |
|------|------|------|
| 缓存方式 | NodeCache 内存（5 分钟 TTL） | 不变 |
| 持久化 | **无** | SQLite 历史快照 |
| 历史查询 | 不支持 | `GET /api/hot/:platform/history?hours=24` |
| 数据生命周期 | 刷新即丢弃 | 保留 7 天，自动清理 |

| 新增依赖 | — | `better-sqlite3` |
| 关键词过滤 | **无** | AI 关键词过滤器，社交平台共用 |

### 8.4 前端表现

| 维度 | V1.0 | V2.0 |
|------|------|------|
| 设计风格 | 炫彩科技风（渐变卡片头部、3D 悬浮） | **深空蓝+香槟金+玻璃拟态**（暗色主题） |

| 详情查看 | 无 | **点击弹出 24h 趋势图**（Canvas 绘制） |
| 平台筛选 | 无 | **Tab 切换**（全部 / 社交平台 / AI 原生） |
| 数据刷新 | 手动刷新页面 | **3 分钟自动轮询** |
| 加载状态 | 无 | 骨架屏 shimmer |
| 品牌色 | 3 种（橙/靛蓝/—） | **5 种**（橙/靛蓝/粉/黄/灰） |
