## 3. 架构变更

### 3.1 新增数据库

V1.0 仅用 NodeCache 内存缓存，V2.0 引入轻量数据库存储历史快照：

```
┌─────────────┐    ┌──────────────┐    ┌───────────┐
│ 适配器采集   │ → │ NodeCache    │ → │ API 响应   │
│ (不变)       │    │ (5min TTL)   │    │ (不变)     │
└─────────────┘    └──────┬───────┘    └───────────┘
                          │ 每次采集后写入
                   ┌──────▼───────┐
                   │ SQLite       │
                   │ (历史快照)    │
                   │ 按天分表      │
                   └──────────────┘
```

- **SQLite** 替代 PostgreSQL（MVP 阶段零运维，单文件存储）
- 每平台每天一张快照表，自动清理 7 天前数据
- 历史趋势 API：`GET /api/hot/:platform/history?hours=24`

### 3.2 新增平台适配器

```
backend/src/adapters/
├── weibo.ts       (已有)
├── zhihu.ts       (已有)
├── baidu.ts       (已有)
├── bilibili.ts    (新增)
└── douyin.ts      (新增)
```

每个适配器遵循相同的 `PlatformAdapter` 接口，零耦合。

## 4. 非功能需求

| 类别 | V1.0 | V2.0 目标 |
|------|------|-----------|
| 性能 | 首页加载 < 2s | 保持，趋势图请求 < 1s |
| 移动端 | 无 | Lighthouse Mobile ≥ 80 |
| 可用性 | 单平台失效不影响其他 | 保持 |
| 数据存储 | 无 | SQLite 单文件 < 100MB |
| 新增平台成本 | 1 个适配器文件 | 保持 |
